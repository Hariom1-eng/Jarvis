# UI Design

## Visual Direction

AntiGravity Jarvis should feel like a premium control room, not a generic chatbot app.

Core visual language:

- glassmorphism panels
- holographic cyan-blue energy accents
- dark graphite background
- subtle amber warnings for caution states
- volumetric glow around assistant presence
- animated waveform, radar, and system telemetry motifs

## Main Screens

### 1. Command Center

Primary dashboard with:

- central avatar viewport
- conversation panel

- quick action chips
- active mode indicator
- current emotional state ring

### 2. Floating Assistant Panel

Compact always-available overlay:

- wake/listen status

### 3. Memory and Identity Screen

- known preferences

- privacy controls

## Avatar Art Direction

Choose one of these product directions:

### Option A: Holographic Human

- realistic human face
- blue emissive contour lines
- floating particles
- soft scanline shimmer

### Option B: Sci-Fi Synthetic

- premium android aesthetic
- metallic facial details
- controlled eye glow
- cleaner futuristic silhouette

### Option C: Lightform Intelligence

- semi-abstract hologram body
- easier to animate well
- more forgiving on mid-range hardware

Option C is the fastest path to a polished MVP. Option A is the highest emotional payoff.

## Animation States

Mandatory states:

- boot greeting
- idle breathing
- listening
- thinking
- speaking
- caution
- celebration
- concern
- sleep

## Expression Mapping

Map assistant emotions to visible states:

- happy: warm cyan with slight smile and bright eyes
- sad: dim blue, softer brows, slowed motion
- thinking: violet-blue analytic pulse, eye focus shift
- serious: stable white-blue glow, reduced motion
- excited: brighter glow and stronger hand emphasis
- confused: head tilt and reduced certainty pulse

## Voice Visualization

Add a live waveform around the avatar base or behind the conversation panel. It should:

- expand with speech amplitude
- shift color with emotion
- become sharper during warnings

## UX Rules

- the assistant should never feel frozen
- every wait state should have a subtle animation
- important actions should have cinematic micro-transitions
- avoid clutter; focus attention on the assistant presence
